Crime Tracking & Analysis API — Role Access Matrix
Base URL prefix: /api/v1
======================================================

USER ROLES
----------
  Viewer  – Self-registered public user. Read-only access to all domain data.
  Officer – Police officer working on cases. Full field operations (create/update cases,
            evidence, suspects, witnesses, victims). Cannot manage trials or punishments.
  Judge   – Judicial authority. Can read all case data and manage trials, hearings,
            punishments, and judge assignments. Cannot open/close cases or manage field data.
            Role value in the database: 'judge'.
  Admin   – Full access to all endpoints.

Legend: ✓ = access granted   ✗ = access denied
======================================================

======================================================
SUMMARY BY ROLE
---------------
Viewer  : Auth (public) + all GET/read endpoints. No write access anywhere.
Officer : Everything Viewer can do + create/update cases, evidence, suspects,
          witnesses, victims, persons, addresses, and officer assignments.
          Cannot create trials, record hearings, assign punishments, or assign judges.
Judge   : Everything Viewer can do + full trial management (create trials, record
          hearings, assign punishments, assign judges). Cannot open/close cases
          or manage field data (evidence, suspects, witnesses, victims).
Admin   : Unrestricted access to all endpoints.
