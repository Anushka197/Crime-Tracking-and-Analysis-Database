Crime Tracking & Analysis API — Endpoints Reference
Base URL prefix: /api/v1
======================================================

AUTH
----
POST   /auth/register                                          – Register a new user (role defaults to 'viewer').
POST   /auth/login                                             – Validate credentials and return a signed JWT access token.
POST   /auth/change-password                                   – Change the password for the currently authenticated user.
POST   /auth/access-check                                      – Check whether the given roles/scopes allow access to a specific endpoint.

CASES
-----
POST   /cases                                                  – Open a new case and optionally assign an initial officer.
GET    /cases                                                  – Return a filtered, sorted, paginated list of cases.
GET    /cases/{case_id}                                        – Fetch basic details of a single case.
PATCH  /cases/{case_id}                                        – Partially update case fields (summary, crime_type, status, etc.).
PATCH  /cases/{case_id}/close                                  – Close a case by setting its status to CLOSED and recording the end date.
GET    /cases/{case_id}/details                                – Return a case with all related sub-entities (evidence, suspects, victims, witnesses, trials, testimonies).
POST   /cases/{case_id}/officers/{officer_id}                  – Assign a police officer to a case (idempotent).
DELETE /cases/{case_id}/officers/{officer_id}                  – Remove an officer's assignment from a case.

EVIDENCE
--------
POST   /cases/{case_id}/evidence                               – Add a new evidence item to a case.
GET    /cases/{case_id}/evidence                               – List all evidence items collected for a case.
GET    /evidence/{evidence_id}                                 – Fetch a single evidence item by its ID.
PATCH  /evidence/{evidence_id}                                 – Partially update an evidence record (description, type, location, collection date).

SUSPECTS
--------
POST   /cases/{case_id}/suspects                               – Add a suspect to a case (by existing person_id or inline new person) and link evidence.
GET    /cases/{case_id}/suspects                               – List all suspects linked to a case.
PATCH  /cases/{case_id}/suspects/{suspect_id}                  – Update a suspect's arrest status, physical description, or bail information.
GET    /suspects/{suspect_id}                                  – Fetch a suspect's profile by their person ID.

VICTIMS
-------
POST   /cases/{case_id}/victims                                – Add a victim to a case (by existing person_id or inline new person).
GET    /cases/{case_id}/victims                                – List all victims affected by a given case.
GET    /victims                                                – Global paginated list of all victims with optional name filter.

WITNESSES
---------
POST   /cases/{case_id}/witnesses                              – Add a witness to a case (by existing person_id or inline new person).
GET    /cases/{case_id}/witnesses                              – List all witnesses linked to a case.
POST   /cases/{case_id}/witnesses/{witness_id}/testimony       – Record or update a witness's testimony and the suspects they point to.
GET    /cases/{case_id}/testimonies                            – List all testimony records for a case.

TRIALS
------
POST   /cases/{case_id}/trials                                 – Create a new trial for a case, with optional judge and hearing date.
GET    /cases/{case_id}/trials                                 – List all trials associated with a case.
GET    /cases/{case_id}/trials/{trial_id}                      – Fetch a trial's full details including all punishment records.
POST   /cases/{case_id}/trials/{trial_id}/hearing              – Record a hearing outcome and notes for a trial.
POST   /cases/{case_id}/trials/{trial_id}/punishment           – Assign punishment (fine, jail, death penalty) to criminals linked to this trial.
POST   /cases/{case_id}/trials/{trial_id}/judge/{judge_id}     – Assign a judge to a specific trial.

PERSONS
-------
POST   /persons                                                – Create a new person (with inline or referenced address).
GET    /persons                                                – Paginated list of persons with optional name and role filters.
GET    /persons/{person_id}                                    – Fetch full details of a person by ID, including all role profiles.
PATCH  /persons/{person_id}                                    – Partially update person details and/or their linked address.
GET    /persons/{person_id}/cases                              – Return all cases associated with a person and the role(s) they hold in each.

ADDRESSES
---------
POST   /addresses                                              – Create a new address record.
GET    /addresses                                              – Return a paginated list of addresses with optional city/country filters.
GET    /addresses/{address_id}                                 – Fetch a single address by ID.
PATCH  /addresses/{address_id}                                 – Partially update an address record.

ANALYTICS
---------
GET    /cases/{case_id}/snapshot                               – Return a combined snapshot of a case's evidence, witnesses, and suspects.
GET    /analytics/hotspots                                     – Aggregate case counts by city and return locations sorted by crime frequency.
